Executive Summary

panos_backup is a complete PAN-OS backup and restore solution for AAP 2.5, built by mlowcher to manage HA firewalls through Panorama. The backup playbook discovers device groups, exports every connected HA member's running config via serial redirection through Panorama, writes timestamped per-group snapshots, prunes to a configurable retention depth, commits to git, and then creates or updates a self-service PANOS Restore job template whose survey dropdowns always reflect current device groups and retained restore points.

Restore loads into candidate config only by default — a deliberate human gate. A commit_and_push survey toggle (added in the third commit) allows hands-off commit with HA config-sync when the operator is confident.

Backup pipeline
discover→ export→ prune→ git push→ update restore JT

Restore survey (self-updating)

The backup playbook rewrites the PANOS Restore template's survey on every run — device group and restore-point dropdowns stay current without manual edits. The restore template appears after the first backup run, not from config-as-code.
